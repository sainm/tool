# APIDiff 使用指南

## 简介

APIDiff 是一个用来比较不同版本 JDK API 差异的工具，由 OpenJDK 项目维护。它可以分析 API 签名变化、文档注释变化，并生成 HTML 格式的比较报告。

## 快速开始

```bash
java -jar /Users/sainm/work/github/tool/lib/apidiff.jar <选项>
```

**前置条件**：运行 apidiff 需要 JDK 17 及以上版本。

---

## 核心概念

APIDiff 一次可以比较多个 API（如 JDK 17、JDK 21、JDK 25），每个 API 通过 `--api <name>` 定义一个名字，然后用一系列 API 专属选项来描述该 API 的 class 来源、文档来源等。

### 最简对比：只比较 API 签名

```bash
java -jar tool/lib/apidiff.jar \
    --api JDK17  --system /usr/lib/jvm/jdk-17 \
    --api JDK21  --system /usr/lib/jvm/jdk-21 \
    --api JDK25  --system /usr/lib/jvm/jdk-25 \
    -d report/
```

这里 `--system` 指向 JDK 安装目录，apidiff 通过 javac API 自动读取 jmod/jrt 中的模块信息。

### 完整对比：签名 + API 描述文档

```bash
java -jar tool/lib/apidiff.jar \
    --api JDK17  --system /usr/lib/jvm/jdk-17  --api-directory /path/to/jdk17/docs/api \
    --api JDK21  --system /usr/lib/jvm/jdk-21  --api-directory /path/to/jdk21/docs/api \
    --api JDK25  --system /usr/lib/jvm/jdk-25  --api-directory /path/to/jdk25/docs/api \
    --compare-api-descriptions true \
    -d report/
```

### 使用 JDK 构建目录（OpenJDK 开发者）

如果你有完整的 JDK 源码和构建产物，`--jdk-build` 会自动推断所有路径：

```bash
java -jar tool/lib/apidiff.jar \
    --api JDK17  --jdk-build /path/to/jdk17/build/ \
    --api JDK21  --jdk-build /path/to/jdk21/build/ \
    --api JDK25  --jdk-build /path/to/jdk25/build/ \
    --compare-api-descriptions true \
    -d report/
```

## 所有选项说明

### 定义 API 的选项（紧跟 `--api` 使用）

| 选项 | 说明 |
|------|------|
| `--api <name>` | 定义 API 名称，后面的 API 专属选项都应用于此 API |
| `--system <path>` | JDK 安装目录（读取 jmod/lib/modules） |
| `--api-directory <path>` | javadoc 输出目录（用于对比文档） |
| `--source-path <path>` | 源码路径 |
| `--module-path <path>` | 模块路径 |
| `--module-source-path <path>` | 模块源码路径 |
| `--class-path <path>` | class 路径 |
| `--patch-module <module>=<path>` | 补丁模块 |
| `--release <version>` | Java SE 发布版本号 |
| `--source <version>` | 源码版本 |
| `--enable-preview` | 启用预览特性 |
| `--label <text>` | 报告中显示的简短标识 |
| `--jdk-build <path>` | JDK 构建目录（自动推断其他选项） |
| `--jdk-docs <name>` | 配合 `--jdk-build`，指定文档包的名称 |

### 比较行为选项（全局）

| 选项 | 说明 |
|------|------|
| `--compare-api-descriptions <bool>` | 是否比较 javadoc 生成的 API 描述 |
| `--compare-api-descriptions-as-text <bool>` | 将 HTML 描述转纯文本后再比较（忽略 HTML 格式差异） |
| `--compare-doc-comments <bool>` | 是否比较源码中的文档注释（需配合 `--source-path`） |
| `--show-unchanged <bool>` | 是否显示未变化的元素（默认只显示有差异的） |

### 过滤选项（全局）

| 选项 | 说明 |
|------|------|
| `--access <level>` | 比较的最低访问级别：`public`（默认）、`protected`、`package`、`private` |
| `--include <pattern>` | 只比较匹配模式的元素 |
| `--exclude <pattern>` | 排除匹配模式的元素 |

### 输出选项（全局）

| 选项 | 说明 |
|------|------|
| `--output-directory <path>` 或 `-d` | 报告输出目录 |
| `--title <text>` | 报告标题 |
| `--description <html>` | 报告简要描述（HTML） |
| `--info-text <position>=<html>` | 在不同位置插入文字，position 可选：`top`、`header`、`footer`、`bottom` |
| `--notes <file>` | 备注文件，内容会嵌入报告中 |
| `--main-stylesheet <file>` | 替代默认样式表 |
| `--extra-stylesheet <file>` | 附加样式表 |
| `--resource-files <path>` | 需要复制的资源文件 |

### 其他

| 选项 | 说明 |
|------|------|
| `--help` 或 `-h` | 显示帮助 |
| `--version` 或 `-v` | 显示版本 |
| `--verbose <flags>` | 详细输出，可选：`all`、`module`、`package`、`type`、`differences`、`missing`、`time`、`options` |

## 命令格式

apidiff 使用**顺序敏感**的参数形式。结构如下：

```
apidiff [全局选项] \
    --api <api1> [api1专属选项...] \
    --api <api2> [api2专属选项...] \
    [全局选项]
```

`--api` 是一个分界符，之前的选项是全局的或属于上一个 API。

## 实际示例

### 示例 1：只比较 API 签名，不看文档

```bash
java -jar tool/lib/apidiff.jar \
    --api JDK17 --system /opt/jdks/jdk-17.0.9.jdk \
    --api JDK21 --system /opt/jdks/jdk-21.0.5.jdk \
    --api JDK25 --system /opt/jdks/jdk-25.jdk \
    --title "JDK 17 → 21 → 25 API 变化" \
    -d report/signature-only
```

### 示例 2：同时比较签名和文档（需要 javadoc）

```bash
java -jar tool/lib/apidiff.jar \
    --api JDK17 \
        --system /opt/jdks/jdk-17 \
        --api-directory /opt/docs/jdk17/api \
    --api JDK21 \
        --system /opt/jdks/jdk-21 \
        --api-directory /opt/docs/jdk21/api \
    --api JDK25 \
        --system /opt/jdks/jdk-25 \
        --api-directory /opt/docs/jdk25/api \
    --compare-api-descriptions true \
    --compare-api-descriptions-as-text true \
    --title "JDK 17/21/25 完整 API 差异报告" \
    -d report/full
```

### 示例 3：只看 public API，加备注

```bash
java -jar tool/lib/apidiff.jar \
    --api JDK17 --system /opt/jdks/jdk-17 \
    --api JDK21 --system /opt/jdks/jdk-21 \
    --access public \
    --title "JDK 17 vs 21 Public API Diff" \
    --info-text "top=本次对比于 2024 年 6 月执行" \
    --notes notes.txt \
    -d report/public-only
```

### 示例 4：使用自定义样式

```bash
java -jar tool/lib/apidiff.jar \
    --api JDK17 --system /opt/jdks/jdk-17 \
    --api JDK21 --system /opt/jdks/jdk-21 \
    --main-stylesheet my-theme.css \
    -d report/styled
```

## 输出

运行后 `-d` 指定的目录下会生成一套完整的 HTML 报告，包括：

```
report/
├── index.html           ← 总览页面
├── <module-name>/       ← 按模块
│   ├── index.html       ← 模块内包列表
│   ├── <package>/
│   │   ├── index.html   ← 包内类型列表
│   │   └── <Type>.html  ← 每个类型的详细差异
│   └── ...
├── apidiff.css          ← 样式
└── element-list         ← 元素列表
```

用浏览器打开 `index.html` 即可查看。

---

## 注意事项

1. 运行 apidiff 的 JDK 版本必须 ≥ 17（建议用 JDK 26 运行本 jar）
2. `lib/` 下 4 个 jar 必须保持在同一目录，不能只拿 apidiff.jar 单独用
3. 如果没有 javadoc 目录，apidiff 只比较 API 签名，不比较文字描述
4. `--system` 支持 JDK 9+ 的 jmod/jrt 格式，无需额外处理
